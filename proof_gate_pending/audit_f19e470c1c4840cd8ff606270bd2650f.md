The `else` branch silently routes `TokenStatus.Staged` (and `Invalid`, `InDuel`, `Killed`) to `PORTAL_TRADE_V2` rather than reverting at the router level. The comment "will fail in implementation" is an assumption, not a guarantee.

## Attacker Path

**preconditions:**
- `SALE_FORGE` has called `stageNewTokenV5` for some token `T`, setting `state.status = TokenStatus.Staged` with `reserve = 0` and `circulatingSupply = 0`. `commitNewTokenV5` has not yet been called, so the token contract at address `T` may not even be deployed yet.
- The attacker is any unprivileged EOA.

**attacker-controlled inputs:**
- `params.inputToken = address(0)` (native BNB, buy direction)
- `params.outputToken = T` (the staged token address)
- `params.inputAmount = some BNB amount`
- `params.minOutputAmount = 0`

**call sequence:**
1. SALE_FORGE calls `stageNewTokenV5(...)` → `T` gets `status = Staged`, `reserve = 0`, `circulatingSupply = 0`.
2. Attacker calls `swapExactInput({inputToken: address(0), outputToken: T, ...})`.
3. `_getSwapImplementation` reads `state.status == TokenStatus.Staged`, falls into the `else` branch, returns `PORTAL_TRADE_V2`.
4. `_delegateToImpl(PORTAL_TRADE_V2)` executes the buy logic in the context of Portal's storage.
5. Whether the call succeeds or reverts depends entirely on whether `PORTAL_TRADE_V2` checks `state.status == TokenStatus.Tradable` before proceeding. [2](#0-1) [3](#0-2) 

## Why Existing Checks Fail

`_getSwapImplementation` performs no status-gate revert for non-`Tradable`, non-`DEX` statuses. The only protection is the implicit assumption that `PORTAL_TRADE_V2` will reject the call. The `TokenNotTradable` error is defined in the interface [4](#0-3)  but whether `PORTAL_TRADE_V2` actually emits it for `Staged` tokens cannot be confirmed from this repository — the `PORTAL_TRADE_V2` implementation source is not present (only the ABI artifact at `out/IPortal.sol/IPortalTradeV2.json`).

If `PORTAL_TRADE_V2` does check `state.status == TokenStatus.Tradable` and reverts, the bug is a defense-in-depth gap (no router-level gate) but not exploitable. If it does not check status, the bonding curve executes against `reserve = 0` and `circulatingSupply = 0`. Standard bonding curve math with zero reserve would likely revert on division-by-zero or produce `outputAmount = 0` (failing `minOutputAmount` if non-zero), but this must be verified empirically. [5](#0-4)
