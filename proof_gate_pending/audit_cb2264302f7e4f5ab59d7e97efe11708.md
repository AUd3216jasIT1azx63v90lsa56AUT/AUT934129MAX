## Attacker Path
**preconditions:**
- The PORTAL_ROLLER implementation does not enforce `ROLLER_ROLE` or `DEFAULT_ADMIN_ROLE` on `msg.sender` inside its `delegateClaim` handler (the question's stated precondition).
- Attacker has previously set or controls `_tokenBeneficiaries[token]` (e.g., by being the original token creator/beneficiary, or by exploiting `setTokenBeneficiary` if accessible).
- The target token has accrued LP fees claimable via the roller.

**attacker-controlled inputs:**
- `token` — any launched token address with a non-zero beneficiary and accrued LP fees.

**call sequence:**
1. Attacker calls `Portal.delegateClaim(token)` from an arbitrary EOA.
2. Portal dispatcher checks only `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)` — passes if protocol is live.
3. `_delegateToImpl(PORTAL_ROLLER)` executes a `delegatecall` into the roller implementation.
4. If the roller has no caller role check, the full claim flow executes, collecting LP fees and sending them to `_tokenBeneficiaries[token]`.

## Why Existing Checks Fail
The Portal.sol dispatcher applies **only** `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)` to `delegateClaim` — there is no `onlyRole(ROLLER_ROLE)` or `onlyRole(DEFAULT_ADMIN_ROLE)` guard at the dispatcher level. [1](#0-0) 

The NatSpec in `IPortal.sol` states "Only the roller or default admin can call this function," but this is documentation only — no on-chain enforcement exists at the Portal layer. [2](#0-1) 

There is **no `ROLLER_ROLE` constant defined anywhere** in the indexed source (`src/Portal.sol`, `src/PortalBase.sol`, `src/interfaces/IPortal.sol`). The only place a role check could exist is inside the PORTAL_ROLLER implementation contract, which is **not present in this repository** — `src/PortalRoller*.sol` does not exist in the repo. [3](#0-2) 

## Rejection Checks

**unsupported assumption checked:** The entire finding rests on whether the PORTAL_ROLLER implementation omits the caller check. This cannot be confirmed from source alone since the roller source is absent from the repo.
