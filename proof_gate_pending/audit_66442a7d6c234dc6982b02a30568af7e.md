## Attacker Path

**preconditions:**
- A V2+ token (`TOKEN_V2_PERMIT`, `TOKEN_TAXED`, `TOKEN_TAXED_V2`, `TOKEN_TAXED_V3`, or `TOKEN_V3_PERMIT`) is in `Tradable` (bonding-curve) status.
- The sell path in `PORTAL_TRADE_V2` (the delegatecall target for `swapExactInput`) passes `payer = address(this)` to `_burnToken` — either always, or when `permitData` is empty/absent and Portal already holds a sufficient token balance.
- Attacker has no tokens of their own.

**attacker-controlled inputs:**
- Direct ERC-20 `transfer(portalAddress, amount)` of the target token to Portal before the swap call.
- `swapExactInput(ExactInputParams{inputToken: token, outputToken: address(0), inputAmount: amount, minOutputAmount: 0, permitData: ""})`.

**call sequence:**
1. Attacker calls `token.transfer(portalProxy, 1000e18)` — donates tokens to Portal.
2. Attacker calls `portal.swapExactInput(...)` with `inputToken = token`, `outputToken = address(0)`, `inputAmount = 1000e18`.
3. Portal dispatches via `_delegateToImpl(PORTAL_TRADE_V2)`.
4. `PORTAL_TRADE_V2` sell logic calls `_burnToken(token, 1000e18, address(this))`.
5. Because `payer == address(this)`, `safeTransferFrom` is skipped; Portal already holds the tokens from step 1.
6. `circulatingSupply` is decremented; bonding-curve math returns BNB to the caller.
7. Attacker receives BNB without having legitimately sold any tokens. [3](#0-2) [4](#0-3) 

## Why Existing Checks Fail

The `_burnToken` guard is purely `payer != address(this)` — there is no secondary check that Portal's token balance increase was funded by `msg.sender` in the same transaction. [5](#0-4) 

The `sell` entrypoint is disabled (`revert FeatureDisabled()`), but `swapExactInput` is live and routes bonding-curve sells to `PORTAL_TRADE_V2` via `delegatecall`. [6](#0-5) [7](#0-6) 

The `ExactInputParams` struct includes a `permitData` field, suggesting the intended design is: permit → pull tokens to Portal → call `_burnToken(…, address(this))`. The critical unknown is whether `PORTAL_TRADE_V2` also falls through to `payer = address(this)` when `permitData` is empty, or whether it always uses `payer = msg.sender` in the no-permit branch. The `PORTAL_TRADE_V2` implementation contract is **not present** in the verified source files (`src/Portal.sol`, `src/PortalBase.sol`, `src/PortalCommon.sol` and interfaces/libraries are the only indexed files). [8](#0-7) 

## Rejection Checks

**expected behavior checked:** The `payer == address(this)` branch is explicitly documented as a gas-saving optimization for the permit-pre-transfer flow. Whether it is also reachable without a prior legitimate pull is the open question.

**prior report checked:** No prior report found in indexed source.

**README/NatSpec checked:** NatDoc on `_burnToken` says "transfer token from the payer to this contract" — it does not document any invariant that Portal's balance must have been funded by `msg.sender` in the same call. [9](#0-8) 

**unsupported assumption checked:** The attack requires `PORTAL_TRADE_V2` to pass `payer = address(this)` on the sell path. This is unconfirmed without that contract's source. If `PORTAL_TRADE_V2` always passes `payer = msg.sender` in the non-permit branch, the attack fails because `safeTransferFrom(attacker, portal, amount)` would revert on insufficient balance.

```text
    /// @dev "burn" token
    ///    for tokenVersion < V2, we burn the token through the token contract
    ///    for tokenVersion >= V2, we transfer token from the payer to this contract
    function _burnToken(address token, uint256 amount, address payer) internal {
        PackedTokenStateV2 memory state = _getTokenState(token);

        if (
            state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_LEGACY_MINT_NO_PERMIT
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_LEGACY_MINT_NO_PERMIT_DUPLICATE
        ) {
            IToken(token).burn(payer, amount);
            state.circulatingSupply = uint128(IToken(token).totalSupply());
        } else if (
            state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_V2_PERMIT
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_TAXED
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_TAXED_V2
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_TAXED_V3
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_V3_PERMIT
        ) {
            // gas saving if the token has already been transferred to this contract before calling _burnToken
            // we don't need to do a self transfer in this case.
            if (payer != address(this)) {
                IERC20(token).safeTransferFrom(payer, address(this), amount);
            }
            state.circulatingSupply -= uint128(amount);
        } else {
            revert NotImplemented();
        }

        // flush state
        _setTokenCirculatingSupply(token, state.circulatingSupply);
        // emit circulation update
        emit FlapTokenCirculatingSupplyChanged(token, uint256(state.circulatingSupply));
    }
```

```text
    function _getSwapImplementation(address inputToken, address outputToken)
        internal
        view
        returns (address implementation)
    {
        // Determine if this is a buy or sell
        bool isBuy = _quoteTokenConfigurations[inputToken].enabled == 1;
        address baseToken = isBuy ? outputToken : inputToken;

        // Get token state to check status
        PackedTokenStateV2 memory state = _getTokenState(baseToken);

        if (state.status == TokenStatus.DEX) {
            // Token is listed on DEX, use PortalDexRouter
            return PORTAL_DEX_ROUTER;
        } else if (state.status == TokenStatus.Tradable) {
            // Token is still in bonding curve, use PortalTradeV2
            return PORTAL_TRADE_V2;
        } else {
            // Invalid or other status, will fail in implementation
            return PORTAL_TRADE_V2; // Let PortalTradeV2 handle the error
        }
    }
```
