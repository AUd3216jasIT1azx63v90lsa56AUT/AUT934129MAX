The Portal.sol entry point carries **no role modifier, no `onlyRole`, no `hasRole` guard, and no `onlyIfBitFlagsSet` check**. Every other privileged tweak function (`setQuoteTokenConfiguration`, `registerExtension`, `setFeeExemption`, etc.) also delegates bare to `PORTAL_TWEAK` — role enforcement is expected to live entirely inside the delegatecall target. [1](#0-0) 

The `PORTAL_TWEAK` implementation address is an immutable set at construction time and is **not present in the indexed source files** for this repository. The verified source inventory (`src/Portal.sol`, `src/PortalBase.sol`, `src/PortalCommon.sol`, interfaces, libraries) does not include a `PortalTweak.sol`. [2](#0-1) 

`AUDITOR_ROLE` appears 10 times in `IPortal.sol` (interface only) and **zero times** in any concrete `.sol` source file in the repo, meaning the role constant definition and its enforcement checks are exclusively in the unindexed `PORTAL_TWEAK` implementation. [3](#0-2) 

## Attacker Path
**preconditions:** PORTAL_TWEAK's `recoverStuckDividend` implementation omits or has a bypassable `AUDITOR_ROLE` check; at least one dividend contract holds live user balances.

**attacker-controlled inputs:** `RecoverStuckDividendParams[]` array — specifically the `taxToken` (pointing to a live dividend-holding contract) and `recipient` (attacker-controlled address).

**call sequence:**
1. Unprivileged EOA calls `Portal.recoverStuckDividend([{taxToken: <live_dividend_contract>, recipient: <attacker>}])`
2. Portal delegates to `PORTAL_TWEAK` with full Portal storage context
3. If PORTAL_TWEAK lacks the role check, it calls `emergencyWithdraw` (or equivalent) on the dividend contract, transferring tokens to the attacker

## Why Existing Checks Fail
At the Portal.sol layer there is provably **no access control** on `recoverStuckDividend`. [1](#0-0)  Compare with `setBitFlags` which explicitly uses `onlyRole(DEFAULT_ADMIN_ROLE)` at the Portal layer. [4](#0-3)  The entire security boundary is delegated to PORTAL_TWEAK, whose source is not available for static verification in this index.
