## Attacker Path
**preconditions:**
- Protocol is live (global switch bit set, i.e. `bitFlags & 1 == 1`)
- At least one token has been launched to DEX with accumulated LP fees
- `_tokenBeneficiaries[token]` is set to any address (attacker does not need to control it to trigger the claim; the fees go to the stored beneficiary regardless)

**attacker-controlled inputs:**
- `token` — any launched token address with claimable LP fees

**call sequence:**
1. Attacker (arbitrary EOA) calls `Portal.delegateClaim(token)`
2. `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)` passes (protocol is live)
3. `_delegateToImpl(PORTAL_ROLLER)` executes the roller implementation in Portal's storage context
4. If the roller implementation has no caller role check → full claim flow executes, LP fees sent to `_tokenBeneficiaries[token]`

## Why Existing Checks Fail

The Portal.sol dispatcher applies **only** `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)` to `delegateClaim` — no `onlyRole`, no `hasRole` check. [1](#0-0) 

The NatSpec on `IRoller.delegateClaim` states "Only the roller or default admin can call this function" — but this is documentation only; enforcement must live inside the PORTAL_ROLLER implementation. [2](#0-1) 

Critically, **there is no `ROLLER_ROLE` constant defined anywhere in the indexed source** (`src/Portal.sol`, `src/PortalBase.sol`, `src/PortalCommon.sol`). The only roles defined are `GUARDIAN_ROLE`, `TAX_MANAGER_ROLE`, `TAX_GUARDIAN_ROLE`, `TOKEN_FLAP_FEE_SETTER_ROLE`, `MODERATOR_ROLE`, and `DEFAULT_ADMIN_ROLE`. [3](#0-2) 

The PORTAL_ROLLER implementation source is **not present** in this repository. Whether it enforces a caller check is unverifiable from the indexed code alone. If it does not, any EOA can trigger the claim flow.

## Rejection Checks
**expected behavior checked:** NatSpec documents the restriction; whether it is implemented is the open question.
**prior report checked:** No prior report found in indexed source.
**README/NatSpec checked:** NatSpec says "roller or default admin only" — but this is in the interface file, not enforced at the dispatcher.
**unsupported assumption checked:** The entire finding is conditional on the roller implementation lacking a caller check. This cannot be confirmed or denied without the roller source or a fork test.
