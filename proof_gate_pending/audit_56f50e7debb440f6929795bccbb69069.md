## Attacker Path
**preconditions:**
- A token is in `TokenStatus.Tradable` with `circulatingSupply` just below `dexSupplyThresh`
- The migration sequence inside `PORTAL_TRADE_V2` calls `luanchToDEX` (external call to migrator) **before** calling `_setTokenStatus(TokenStatus.DEX)` — this ordering is the unverified assumption
- The migrator or an intermediate external call (e.g., a token transfer hook, a V3 pool callback, or a BNB `receive()` on the migrator) can be intercepted by an attacker-controlled contract

**attacker-controlled inputs:**
- Attacker deploys a contract that receives the external call during `luanchToDEX` and re-enters `swapExactInput` with a buy for the same token

**call sequence:**
1. Attacker calls `swapExactInput` (buy) with exactly enough input to push `circulatingSupply >= dexSupplyThresh`
2. Inside `PORTAL_TRADE_V2` (delegatecall context): buy executes, migration is triggered
3. `PORTAL_TRADE_V2` calls `luanchToDEX` on the migrator — external call, reentrancy window opens
4. Migrator calls attacker contract (e.g., via BNB transfer or token callback)
5. Attacker re-enters `swapExactInput` (buy) for the same token
6. `_getSwapImplementation` reads `state.status` — if still `Tradable` (because `_setTokenStatus(DEX)` has not yet been called), routes to `PORTAL_TRADE_V2`
7. Attacker buys tokens at bonding-curve price; tokens are minted/transferred to attacker
8. Outer call resumes: `_setTokenStatus(DEX)` is now called, DEX pool is funded — but attacker already holds tokens bought at pre-migration bonding-curve price [4](#0-3) [5](#0-4) 

## Why Existing Checks Fail

**No reentrancy guard anywhere in available source.** `grep` for `nonReentrant`, `ReentrancyGuard`, `_ENTERED`, `_NOT_ENTERED` returns zero matches across all `.sol` files in `src/`. `swapExactInput` carries only `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)`, which is a circuit-breaker flag check, not a reentrancy lock. [2](#0-1) 

**The routing check is not a reentrancy guard.** `_getSwapImplementation` reads `state.status` from storage at the moment of the reentrant call. It only blocks the attack if `_setTokenStatus(DEX)` has already been committed to storage before the external call to `luanchToDEX`. If the ordering inside `PORTAL_TRADE_V2` is `luanchToDEX` → `_setTokenStatus(DEX)`, the status is still `Tradable` during the external call and the routing check passes the reentrant buy through to `PORTAL_TRADE_V2`. [1](#0-0) [3](#0-2) 

**Critical unverified assumption:** The `PORTAL_TRADE_V2` implementation is an immutable address set in `PortalBase`; its source is not present in this repository. The exact ordering of `_setTokenStatus(DEX)` vs `luanchToDEX` within the buy/migration path cannot be confirmed from available source. If `_setTokenStatus(DEX)` is called **first**, the routing check in `_getSwapImplementation` would redirect the reentrant call to `PORTAL_DEX_ROUTER` and the attack fails. The entire finding pivots on this ordering. [3](#0-2) 

## Rejection Checks
**expected behavior checked:** Buying at bonding-curve price after migration is triggered but before the DEX pool is funded is not documented as expected behavior; the `TokenStatus.DEX` transition is intended to be atomic with liquidity provision. [6](#0-5) 

**prior report checked:** No prior report found in available source or NatSpec. 

**README/NatSpec checked:** `IPortalMigrator` NatSpec states `luanchToDEX` "may be delegated called from a payable function" but gives no ordering guarantee relative to status update. [7](#0-6) 

**unsupported assumption checked:** The attack requires the migrator to make an external call that the attacker can intercept (e.g., BNB transfer to attacker-controlled address, V3 pool mint callback, or token transfer hook). This is plausible for V3/V4 migrators that call `mint` callbacks, but requires confirmation from the migrator implementation.

    function _getSwapImplementation(address inputToken, address outputToken)
        internal
        view
        returns (address implementation)
    {
        // Determine if this is a buy or sell
        bool isBuy = _quoteTokenConfigurations[inputToken].enabled == 1;
        address baseToken = isBuy ? outputToken : inputToken;

        // Get token state to check status
        PackedTokenStateV2 memory state = _getTokenState(baseToken);

        if (state.status == TokenStatus.DEX) {
            // Token is listed on DEX, use PortalDexRouter
            return PORTAL_DEX_ROUTER;
        } else if (state.status == TokenStatus.Tradable) {
            // Token is still in bonding curve, use PortalTradeV2
            return PORTAL_TRADE_V2;
        } else {
            // Invalid or other status, will fail in implementation
            return PORTAL_TRADE_V2; // Let PortalTradeV2 handle the error
        }
    }

    /// @dev set the token status
    function _setTokenStatus(address token, IPortalTypes.TokenStatus status) internal {
        if (token != address(uint160(uint256(uint160(token))))) revert DirtyBits();
        uint256 slot;
        assembly ("memory-safe") {
            mstore(0x0, token)
            mstore(0x20, _packedTokenStates.slot)
            slot := keccak256(0x0, 0x40)
        }
        uint256 packed;
        assembly ("memory-safe") {
            packed := sload(slot)
        }
        uint8 header = uint8(packed);
        if (header != PACKED_TOKEN_STATE_HEADER) {
            packed = (packed & ~uint256(0xff)) | uint8(status);
            assembly ("memory-safe") {
                sstore(slot, packed)
            }
        } else {
            packed = (packed & ~(uint256(0xff) << 8)) | (uint256(uint8(status)) << 8);
            assembly ("memory-safe") {
                sstore(slot, packed)
            }
        }
    }
