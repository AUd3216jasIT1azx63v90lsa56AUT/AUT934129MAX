## Attacker Path

**preconditions:**
- A token is in `TokenStatus.Tradable` with `circulatingSupply` just below `dexSupplyThresh`.
- The migration path in `PORTAL_TRADE_V2` makes at least one external call (e.g., to a migrator, router, or token contract) **before** `_setTokenStatus(token, TokenStatus.DEX)` is committed to storage.
- Attacker controls or can influence a contract that receives that external call (e.g., a quote-token ERC20 with a hook, a migrator callback, or a BNB `receive()` on a refund).

**attacker-controlled inputs:**
- `ExactInputParams` with `inputToken` = quote token (or native BNB), `outputToken` = target token, `amountIn` = exactly the remaining amount to push `circulatingSupply` to `dexSupplyThresh`.

**call sequence:**
1. Attacker calls `swapExactInput` with the threshold-crossing amount.
2. `PORTAL_TRADE_V2` processes the buy, detects `circulatingSupply >= dexSupplyThresh`, and begins migration.
3. Migration makes an external call (migrator callback, token transfer, BNB send, etc.) **before** `_setTokenStatus(DEX)` is written.
4. Attacker's contract re-enters `swapExactInput` with a new buy.
5. `_getSwapImplementation` reads storage: status is still `Tradable` → routes to `PORTAL_TRADE_V2`.
6. `PORTAL_TRADE_V2` processes the second buy at bonding-curve price (reserve/supply not yet updated for DEX migration).
7. Attacker receives tokens at pre-migration bonding-curve price; DEX pool is subsequently funded with a depleted reserve.

## Why Existing Checks Fail

The only guard against post-threshold trading is the status check inside `_getSwapImplementation` (and presumably inside `PORTAL_TRADE_V2` itself). Both checks read from `_packedTokenStates` storage. If `_setTokenStatus(DEX)` has not yet been written when the reentrant call arrives, both checks see `Tradable` and permit the trade.

There is **no reentrancy lock** anywhere in the indexed source. The `_delegateToImpl` assembly trampoline at lines 831–843 of `PortalBase.sol` contains no mutex. [4](#0-3) 

The `onlyIfBitFlagsSet` modifier on `swapExactInput` only checks the global halt flag — it is not a reentrancy guard. [2](#0-1) 

**Unresolvable without `PORTAL_TRADE_V2` source:** The exact ordering of `_setTokenStatus(DEX)` relative to external calls in the migration path is in `PORTAL_TRADE_V2` and the migrator contracts, which are **not present in the indexed source tree**. The repository contains only `Portal.sol`, `PortalBase.sol`, `PortalCommon.sol`, interfaces, and libraries. 

**Important constraint on the scenario:** If `_setTokenStatus(DEX)` is written *before* any external call, a reentrant `swapExactInput` would be routed to `PORTAL_DEX_ROUTER` (not the bonding curve), and the DEX router would fail because the pool is unfunded — no bonding-curve price extraction occurs. The attack only works if the status write happens *after* an exploitable external call. This ordering question is the crux that requires local proof.

## Rejection Checks

**expected behavior checked:** Buying at bonding-curve price after migration threshold is crossed is not expected behavior; the protocol invariant is that no bonding-curve trades occur after the DEX transition. [5](#0-4) 

**prior report checked:** No prior report found in `SECURITY.md`, `RESEARCHER.md`, or `README.md` addressing this specific reentrancy window. [6](#0-5) 

**README/NatSpec checked:** No NatSpec on `swapExactInput` or `_setTokenStatus` documents a reentrancy-safe ordering guarantee. [7](#0-6) 

**unsupported assumption checked:** The attack requires an external call in the migration path before the status write. This is plausible (migrators routinely call external routers/lockers) but unconfirmed without `PORTAL_TRADE_V2` source.

    function _getSwapImplementation(address inputToken, address outputToken)
        internal
        view
        returns (address implementation)
    {
        // Determine if this is a buy or sell
        bool isBuy = _quoteTokenConfigurations[inputToken].enabled == 1;
        address baseToken = isBuy ? outputToken : inputToken;

        // Get token state to check status
        PackedTokenStateV2 memory state = _getTokenState(baseToken);

        if (state.status == TokenStatus.DEX) {
            // Token is listed on DEX, use PortalDexRouter
            return PORTAL_DEX_ROUTER;
        } else if (state.status == TokenStatus.Tradable) {
            // Token is still in bonding curve, use PortalTradeV2
            return PORTAL_TRADE_V2;
        } else {
            // Invalid or other status, will fail in implementation
            return PORTAL_TRADE_V2; // Let PortalTradeV2 handle the error
        }
    }

        if (impl == address(0)) {
            revert FeatureDisabled();
        }
        assembly {
            calldatacopy(0, 0, calldatasize())
            let result := delegatecall(gas(), impl, 0, calldatasize(), 0, 0)
            returndatacopy(0, 0, returndatasize())
            switch result
            case 0 { revert(0, returndatasize()) }

    function _setTokenStatus(address token, IPortalTypes.TokenStatus status) internal {
        if (token != address(uint160(uint256(uint160(token))))) revert DirtyBits();
        uint256 slot;
        assembly ("memory-safe") {
            mstore(0x0, token)
            mstore(0x20, _packedTokenStates.slot)
            slot := keccak256(0x0, 0x40)

        assembly ("memory-safe") {
            packed := sload(slot)
        }

            packed = (packed & ~uint256(0xff)) | uint8(status);
            assembly ("memory-safe") {
                sstore(slot, packed)
            }
        } else {
            packed = (packed & ~(uint256(0xff) << 8)) | (uint256(uint8(status)) << 8);
            assembly ("memory-safe") {
                sstore(slot, packed)
            }
        }
    }
