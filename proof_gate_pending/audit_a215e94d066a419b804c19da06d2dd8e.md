The Portal.sol wrapper applies **only** `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)` — no role check at the dispatch layer. All access-control enforcement must live inside the `PORTAL_ROLLER` delegatecall target. [1](#0-0) 

## Attacker Path
**preconditions:**
- Protocol is not halted (`CB_BIT_MASK_GLOBAL_SWITCH` bit is set, which is the normal live state).
- A token exists with a non-zero `_tokenBeneficiaries[token]` entry (attacker has previously called `setTokenBeneficiary` or is the original creator whose address was recorded as beneficiary, or the attacker controls the beneficiary address through some other means).
- The `PORTAL_ROLLER` implementation does **not** enforce `ROLLER_ROLE` or `DEFAULT_ADMIN_ROLE` on `msg.sender`.

**attacker-controlled inputs:**
- `token` — any launched token address whose beneficiary the attacker controls.

**call sequence:**
1. Attacker (arbitrary EOA) calls `Portal.delegateClaim(token)`.
2. Portal passes `onlyIfBitFlagsSet` (global switch is on).
3. `_delegateToImpl(PORTAL_ROLLER)` executes the roller's `delegateClaim` logic in Portal's storage context.
4. If the roller has no caller check, it collects LP fees and sends them to `_tokenBeneficiaries[token]`. [2](#0-1) [3](#0-2) 

## Why Existing Checks Fail
The Portal.sol dispatch layer has **zero role enforcement** on `delegateClaim` — only the global-switch bit flag. The IPortal.sol NatSpec states "Only the roller or default admin can call this function," but this is documentation only; no `onlyRole` modifier or equivalent is applied before the delegatecall. [4](#0-3) 

Critically, **no `ROLLER_ROLE` constant is defined anywhere in the indexed source** (`src/Portal.sol`, `src/PortalBase.sol`, `src/PortalCommon.sol`). The grep for `ROLLER_ROLE` returns zero matches in `.sol` files. This means either the roller implementation uses a different guard (e.g., `DEFAULT_ADMIN_ROLE`, a hardcoded address check, or `msg.sender == address(this)`), or it has no guard at all. The `PORTAL_ROLLER` implementation contract source is **not present in this repository** and cannot be verified from the indexed code. [5](#0-4) 

## Rejection Checks
**expected behavior checked:** NatSpec explicitly documents the restriction, but the Portal.sol wrapper does not enforce it — the restriction is delegated entirely to the roller implementation which is absent from the repo.

**prior report checked:** No prior report found in the indexed source.

**unsupported assumption checked:** The core assumption — that the roller implementation lacks a caller check — cannot be confirmed or denied without the roller source. This is the blocking uncertainty.
