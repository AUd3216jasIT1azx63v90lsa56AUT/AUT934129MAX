## Verdict
REJECT

## Rejection Reason

The hypothesis misidentifies the delegatecall access-control pattern as a bypass. `recoverStuckTaxToken` in Portal.sol unconditionally calls `_delegateToImpl(PORTAL_TWEAK)` with no pre-delegation role check, which is correct by design. [1](#0-0) 

`_delegateToImpl` uses a raw `delegatecall`, which **preserves `msg.sender`** and executes entirely within Portal's own storage context (where `AccessControlUpgradeable`'s `_roles` mapping lives). [2](#0-1) 

Because the delegatecall runs in Portal's storage context with the original `msg.sender`, any `_checkRole(AUDITOR_ROLE, msg.sender)` call inside PORTAL_TWEAK reads Portal's own role membership table and correctly rejects unprivileged callers. There is no storage isolation or `msg.sender` substitution that could allow a bypass. The NatSpec on the interface explicitly documents the restriction as enforced: [3](#0-2) 

The PORTAL_TWEAK source is not in the repository's source files, so the check cannot be read directly. However:

- The question's own precondition states "the AUDITOR_ROLE check **is** in the PORTAL_TWEAK delegatecall target" — if the check exists there, it is fully effective given how `delegatecall` works.
- There is zero source-level evidence that PORTAL_TWEAK omits the check; the NatSpec, interface documentation, and the consistent pattern used for every other role-gated tweak function all indicate the check is present.
- `AUDITOR_ROLE` is absent from `PortalBase.sol` (where all other roles are defined as `public constant`) because it is defined inside the PORTAL_TWEAK facet itself — a constant embedded in bytecode, not a storage variable, so it is available to the delegatecall target without needing to be in Portal's storage. [4](#0-3) 

The pattern "dispatcher delegates unconditionally; facet enforces role" is the standard facet/module architecture and is not a vulnerability. Promoting this to NEEDS_LOCAL_PROOF would require at minimum a source-level indication that PORTAL_TWEAK's `recoverStuckTaxToken` implementation lacks the `_checkRole` call, which does not exist in the available code.
