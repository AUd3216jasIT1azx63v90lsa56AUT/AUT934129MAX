## Attacker Path

**preconditions:**
- A legacy-curve token (h=0, `k = r * TOTAL_SUPPLY` via `LibCurve.fromR`) with status `Tradable` exists on the proxy.
- The token implementation used by Portal has a transfer hook (e.g. ERC777-style `_afterTokenTransfer`, or a custom hook in one of Portal's own token implementations) that can call back into Portal.
- No reentrancy guard exists anywhere in the Portal source tree (confirmed: `grep_search` for `nonReentrant|ReentrancyGuard` returns zero matches across all `src/**/*.sol`).

**attacker-controlled inputs:**
- `swapExactInput` with `inputToken = address(0)` (native BNB), `outputToken = <legacy token>`, `inputAmount` sized to push `circulatingSupply` to exactly `dexSupplyThresh`.

**call sequence:**
1. Attacker calls `Portal.swapExactInput(buy params)`.
2. Portal delegates to `PORTAL_TRADE_V2` (via `_delegateToImpl`).
3. `PORTAL_TRADE_V2` computes the buy amount, calls `_mintToken(token, amount, attacker)`.
4. Inside `_mintToken`, for TOKEN_V2_PERMIT+ tokens: `IERC20(token).safeTransfer(attacker, amount)` fires **before** `_setTokenCirculatingSupply` writes to storage.
5. If the token has a transfer hook, the hook re-enters `Portal.swapExactInput` with another buy.
6. The re-entrant call reads stale storage: `circulatingSupply` and `reserve` have not yet been updated, so the curve still prices tokens at the pre-migration level.
7. The re-entrant buy succeeds at the stale (lower) price; `_setTokenStatus(DEX)` has not yet been committed.
8. Both calls eventually write back, but the reserve accounting is corrupted: the proxy has received less BNB than the curve requires for the tokens minted.

## Why Existing Checks Fail

**No reentrancy guard:** `grep_search` for `nonReentrant`, `ReentrancyGuard`, `_reentrancyGuard`, `_status`, `_locked` returns **zero matches** in all `src/**/*.sol` files. [1](#0-0) 

**CEI violation in `_mintToken`:** The external token transfer (`IERC20(token).safeTransfer`) is executed on line 763 **before** the storage write for `circulatingSupply` on line 770 (`_setTokenCirculatingSupply`). A reentrant call during the transfer reads the old on-chain `circulatingSupply`, allowing it to compute a stale (pre-migration) price. [2](#0-1) 

**Legacy token path also vulnerable:** For `TOKEN_LEGACY_MINT_NO_PERMIT`, `IToken(token).mint(recipient, amount)` is an external call on line 754, and `state.circulatingSupply` is only refreshed from `totalSupply()` after it returns â€” same window. [3](#0-2) 

**`_getSwapImplementation` reads live state:** The routing check at `Portal.swapExactInput` re-reads `_getTokenState(baseToken).status` on every entry. If `_setTokenStatus(DEX)` has not yet been committed when the re-entrant call arrives, the router still dispatches to `PORTAL_TRADE_V2` (bonding-curve path) rather than `PORTAL_DEX_ROUTER`. [4](#0-3) 

**Unknown ordering in `PORTAL_TRADE_V2`:** The actual buy logic â€” including the dexSupplyThresh check, reserve update, and `_setTokenStatus(DEX)` call â€” lives in the `PORTAL_TRADE_V2` delegatecall target, which is **not present** in the indexed source files. If that implementation updates the reserve and status *after* calling `_mintToken`, the full reentrancy window is open. If it updates them *before*, the attack is blocked. This ordering is the decisive unknown.

## Rejection Checks

**expected behavior checked:** CEI violation in `_mintToken` is not documented as intentional; no NatSpec or comment acknowledges the reentrancy risk. [5](#0-4) 

**prior report checked:** No prior report reference found in the indexed source.

**README/NatSpec checked:** No NatSpec on `_mintToken` or `swapExactInput` acknowledges reentrancy safety. [6](#0-5) 

**unsupported assumption checked:** The attack requires a token with a transfer hook. Portal's own token implementations (`tokenImplLegacy`, `tokenImplV2`, `tokenImplTaxed`, etc.) are fixed immutables set at deploy time and are **not** in the indexed source. Whether any of them implement `_afterTokenTransfer` or an ERC777-style hook is unconfirmed and is the second decisive unknown.

## Local Proof Required

**test type:** Foundry fork test (BSC mainnet fork at block 108382650) + unit test with mock token.

**test file to add:** `test/ReentrancyBuyMigration.t.sol`

**test setup:**
1. Fork BSC at the live block; use proxy `0xe2ce6ab80874fa9fa2aae65d277dd6b8e65c9de0`.
2. Identify a live legacy-curve token (h=0) with `circulatingSupply` close to `dexSupplyThresh`.
3. Deploy a `MaliciousReceiver` contract whose `onERC20Received` / `_afterTokenTransfer` hook calls `Portal.swapExactInput` again with a small buy.
4. Alternatively, deploy a fresh token via `newTokenV2` using a mock token implementation that has a transfer hook.
5. Record proxy BNB balance before and after the double-buy sequence.

**expected assertion:**
```
assertEq(
    proxy.balance + tokensReceived * postMigrationPrice,
    proxy.balance_before + totalBNBPaid,
    "reserve invariant broken"
);
```
The test should assert that the proxy BNB balance equals `k/(TOTAL_SUPPLY - finalCirculatingSupply) - r` after both calls settle. If the invariant is violated (proxy holds less BNB than the curve requires), the bug is confirmed.

**failure condition:** If `PORTAL_TRADE_V2` updates reserve and status **before** calling `_mintToken`, the re-entrant call will see `status == DEX` and be routed to `PORTAL_DEX_ROUTER`, blocking the attack. The test would then revert or show no balance discrepancy, and the finding should be downgraded to REJECT.

            // Token is still in bonding curve, use PortalTradeV2
            return PORTAL_TRADE_V2;
        } else {
            // Invalid or other status, will fail in implementation
            return PORTAL_TRADE_V2; // Let PortalTradeV2 handle the error
        }
    }

    ///    for tokenVersion < V2, we mint the token through the token contract
    ///    for tokenVersion >= V2, we transfer token from this contract to the recipient
    /// @dev Note! this function changes the state of the token
    function _mintToken(address token, uint256 amount, address recipient) internal {
        PackedTokenStateV2 memory state = _getTokenState(token);

        if (
            state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_LEGACY_MINT_NO_PERMIT
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_LEGACY_MINT_NO_PERMIT_DUPLICATE
        ) {
            IToken(token).mint(recipient, amount);
            state.circulatingSupply = uint128(IToken(token).totalSupply());
        } else if (
            state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_V2_PERMIT
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_TAXED
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_TAXED_V2
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_TAXED_V3
                || state.tokenVersion == IPortalTypes.TokenVersion.TOKEN_V3_PERMIT
        ) {
            IERC20(token).safeTransfer(recipient, amount);
            state.circulatingSupply += uint128(amount);
        } else {
            revert NotImplemented();
        }

        // flush state
        _setTokenCirculatingSupply(token, state.circulatingSupply);

        // emit circulation update
        emit FlapTokenCirculatingSupplyChanged(token, uint256(state.circulatingSupply));
    }
