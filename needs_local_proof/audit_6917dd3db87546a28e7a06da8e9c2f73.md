## Why Existing Checks Fail
The `swapExactInput` entry point in `src/Portal.sol` performs no `msg.value == inputAmount` check before delegating. [3](#0-2) 

The `_delegateToImpl` assembly trampoline forwards the full calldata and the full `msg.value` to the implementation without any pre-validation. [4](#0-3) 

The `ExactInputParams` struct carries both `inputAmount` and the implicit `msg.value`; there is no interface-level constraint requiring them to be equal for native buys. [5](#0-4) 

Notably, `buyOnCreation` explicitly documents that `msg.value > inputAmount` is allowed and the excess is kept as a fee â€” but that NatSpec applies only to `buyOnCreation`, not to `swapExactInput`. [6](#0-5) 

Whether `swapExactInput`'s implementation in `PORTAL_TRADE_V2` enforces `msg.value == inputAmount`, uses `msg.value` for the reserve update, or uses `inputAmount` for the reserve update cannot be determined from the indexed source.

        if (impl == address(0)) {
            revert FeatureDisabled();
        }
        assembly {
            calldatacopy(0, 0, calldatasize())
            let result := delegatecall(gas(), impl, 0, calldatasize(), 0, 0)

            default { return(0, returndatasize()) }
        }
