## Verdict
REJECT

## Rejection Reason

The entire claim rests on the internal `sqrtPriceX96` calculation inside the PCS Infinity CL migrator implementation (`PORTAL_PCS_INFINITY_CL_MIGRATOR`), but that contract's source code is not present in the indexed repository. The only files in scope are `src/Portal.sol`, `src/PortalBase.sol`, `src/PortalCommon.sol`, the interface files, and the two libraries — none of which contain the migrator's pool-initialization logic.

`newTokenV7` in `src/Portal.sol` simply delegates to `PORTAL_LAUNCHER` via `_delegateToImpl(PORTAL_LAUNCHER)`, and `PORTAL_LAUNCHER` in turn delegates to `LAUNCHER_V7_IMPL` — neither of those implementation contracts is in the indexed source. [1](#0-0)  The `PORTAL_PCS_INFINITY_CL_MIGRATOR` immutable is declared and resolved in `PortalBase.sol`, but its implementation file (`PortalPCSInfinityCLMigrator.sol`) is absent from the repository. [2](#0-1)  The `FlapTokenCLPoolCreated` event confirms a `sqrtPriceX96` is emitted at pool creation, but the computation that produces it is entirely inside the missing migrator. [3](#0-2) 

Without the migrator source, the specific claim — that `sqrtPriceX96` is derived from the pre-fee reserve/supply ratio rather than the post-`LIQUIDITY_FEE` amounts actually deposited — is pure speculation. The `LIQUIDITY_FEE` immutable and `_liquidityFeeByProfile` helper exist in `PortalCommon.sol`, confirming a fee is deducted before migration, but whether the migrator correctly adjusts the price seed for that deduction cannot be determined from the available code. [4](#0-3) 

The rules require: "Reject source-only theories that do not connect to the live proxy address, implementation ABI, reachable entrypoint, **concrete value movement**, and local proof plan." The concrete value movement here depends entirely on the migrator's unindexed implementation. Promoting this to `NEEDS_LOCAL_PROOF` or higher without any source-level evidence of the alleged miscalculation would be speculative.

    /// @param profile The fee profile to use
    /// @return The liquidity fee in basis points (bps), where 1% = 100 bps
    function _liquidityFeeByProfile(FlapFeeProfile profile) internal view returns (uint256) {
        if (profile == FlapFeeProfile.FEE_GLOBAL_DEFAULT) {
            return LIQUIDITY_FEE;
        } else if (profile == FlapFeeProfile.FEE_FLAPSALE_V0) {
            return 0; // 0%
        } else if (profile == FlapFeeProfile.FEE_ZERO) {
            return 0; // 0% - no protocol fee
        } else {
            // Unknown profile, default to FEE_GLOBAL_DEFAULT
            return LIQUIDITY_FEE;
        }
    }
