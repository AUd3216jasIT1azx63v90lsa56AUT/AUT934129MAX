## Attacker Path

**preconditions:**
- Attacker is `_tokenBeneficiaries[token]` for a token whose `lpLocks[token]` is non-zero (i.e., a DEX-graduated token with accumulated LP fees)
- Attacker controls a contract with a malicious `receive()` that re-enters `claim(token)`

**attacker-controlled inputs:**
- `token` address passed to `claim()`
- Attacker's beneficiary address is a contract with re-entrant `receive()`

**call sequence:**
1. Attacker (beneficiary contract) calls `Portal.claim(token)`
2. Portal `_delegateToImpl(PORTAL_ROLLER)` — roller runs in Portal's storage context
3. Roller calls locker's `collectFees` → locker sends ETH to Portal (via `Portal.receive()`)
4. Roller then sends ETH to `_tokenBeneficiaries[token]` (attacker contract)
5. Attacker's `receive()` re-enters `Portal.claim(token)` before roller clears the claimable state
6. Second claim executes against the same (not-yet-zeroed) claimable amount
7. Repeat until gas exhaustion or balance drained

## Why Existing Checks Fail

**No reentrancy guard anywhere in Portal:** A comprehensive search for `nonReentrant`, `ReentrancyGuard`, `_reentrancy`, `_locked`, and `_entered` across all Portal source files (`Portal.sol`, `PortalBase.sol`, `PortalCommon.sol`) returned zero matches — only OpenZeppelin library internals matched. [4](#0-3) 

The `_delegateToImpl` assembly trampoline passes all gas and has no guard slot. Since `PORTAL_ROLLER` runs via `delegatecall` in Portal's storage context, any reentrancy guard the roller might declare would occupy a Portal storage slot — but no such slot is reserved or initialized in `PortalBase`'s storage layout. [5](#0-4) 

The `onlyIfBitFlagsSet` modifier only checks feature flags, not reentrancy: [6](#0-5) 

**Critical unknown:** The `PORTAL_ROLLER` implementation contract is not present in this repository. Whether the roller itself implements CEI (clears claimable state before the ETH transfer) or has its own reentrancy guard cannot be confirmed from available source. This is the single blocking uncertainty.

## Rejection Checks

**expected behavior checked:** Claiming LP fees for a beneficiary is a legitimate operation; reentrancy protection is not documented as intentionally absent.

**prior report checked:** No prior report found in available source or NatSpec.

**README/NatSpec checked:** No NatSpec on `claim` mentions reentrancy safety or CEI ordering.

**unsupported assumption checked:** The theory requires the roller to transfer ETH to the beneficiary *before* zeroing the claimable amount. This is the unverifiable assumption — the roller source is not in-repo. If the roller uses CEI correctly, the bug does not exist.

## Local Proof Required

**test type:** Foundry fork test on BSC

**test file to add:** `test/ClaimReentrancy.t.sol`

**test setup:**
1. Fork BSC at block 108382650
2. Identify a token address where `lpLocks[token]` is non-zero (query `getLocks(token)` on the live proxy `0xe2ce6ab80874fa9fa2aae65d277dd6b8e65c9de0`)
3. Impersonate or set `_tokenBeneficiaries[token]` to an attacker contract (requires knowing the current beneficiary or using `vm.store` to overwrite slot 169)
4. Deploy attacker contract whose `receive()` calls `Portal.claim(token)` recursively up to N times
5. Record `Portal.balance` before and after

**expected assertion:**
```
assertEq(portalBalanceBefore, portalBalanceAfter + legitimateFeeAmount);
// i.e., attacker received exactly one fee collection, not N
```

**failure condition:** If `portalBalanceAfter < portalBalanceBefore - legitimateFeeAmount`, the invariant is broken and the bug is confirmed. If the second `claim` call reverts or returns zero, the roller uses CEI or has its own guard and the report should be rejected.

    function _delegateToImpl(address impl) internal {
        if (impl == address(0)) {
            revert FeatureDisabled();
        }
        assembly {
            calldatacopy(0, 0, calldatasize())
            let result := delegatecall(gas(), impl, 0, calldatasize(), 0, 0)
            returndatacopy(0, 0, returndatasize())
            switch result
            case 0 { revert(0, returndatasize()) }
            default { return(0, returndatasize()) }
        }
    }
