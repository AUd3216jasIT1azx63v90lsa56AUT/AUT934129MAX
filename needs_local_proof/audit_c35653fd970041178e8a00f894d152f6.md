## Exact Code Path

file: `src/Portal.sol`
function: `setTokenBeneficiary`
symbols/lines: lines 351–360

```solidity
/// @inheritdoc IRoller
function setTokenBeneficiary(
    address,
    /*token*/
    address /* newBeneficiary */
)
    external
    override
{
    _delegateToImpl(PORTAL_ROLLER);
}
```

The Portal-level entry point carries **no access-control modifier** — no `onlyRole(DEFAULT_ADMIN_ROLE)`, no `onlyIfBitFlagsSet`, no caller check of any kind. [1](#0-0) 

The `_tokenBeneficiaries` mapping that would be mutated lives in `PortalBase` at storage slot 169. [2](#0-1) 

The roller is an immutable facet (`PORTAL_ROLLER`) set at construction time and invoked via `delegatecall`, so it writes directly into Portal's storage. [3](#0-2) 

The PORTAL_ROLLER source file is **not present** in this repository's source tree (the verified source files are `Portal.sol`, `PortalBase.sol`, `PortalCommon.sol`, and the listed interfaces/libraries). Whether the roller's own `setTokenBeneficiary` implementation enforces `hasRole(DEFAULT_ADMIN_ROLE, msg.sender)` or a current-beneficiary check cannot be confirmed from the available indexed code alone. [1](#0-0) 

## Attacker Path

**preconditions:**
- Attacker previously created a token via any `newTokenV*` path, setting themselves as beneficiary in `_tokenBeneficiaries[token]`.
- Token has graduated to DEX (or is still on bonding curve — beneficiary payouts and LP-fee distributions flow to `_tokenBeneficiaries[token]` regardless of phase).
- Attacker does **not** hold `DEFAULT_ADMIN_ROLE`.

**attacker-controlled inputs:**
- `token` = address of the launched token
- `newBeneficiary` = attacker-controlled address B (or any third-party victim address for griefing, but self-redirect is the extraction path)

**call sequence:**
1. Attacker calls `Portal.setTokenBeneficiary(token, attackerAddressB)` with no special role.
2. Portal dispatches `_delegateToImpl(PORTAL_ROLLER)` with no guard.
3. If the roller's implementation also lacks a caller check, it writes `_tokenBeneficiaries[token] = attackerAddressB`.
4. All subsequent `claim`, `delegateClaim`, LP-fee distributions, and beneficiary payouts for `token` are redirected to `attackerAddressB`.

## Why Existing Checks Fail

At the Portal dispatcher level there is no `onlyRole`, no `onlyIfBitFlagsSet`, and no `msg.sender` validation before the `delegatecall`. [1](#0-0)  Compare with `setBitFlags`, which correctly applies `onlyRole(DEFAULT_ADMIN_ROLE)` at the Portal level. [4](#0-3)  The only remaining protection would be inside the roller implementation itself — which is not available for static review in this repository.

## Rejection Checks

**expected behavior checked:** Changing a token beneficiary post-launch to an arbitrary address is not documented as a permissionless action; the invariant stated in `PortalBase` is that `_tokenBeneficiaries` is role-protected state. [2](#0-1) 

**prior report checked:** No prior report found in the indexed source.

**README/NatSpec checked:** No NatSpec on `setTokenBeneficiary` in `Portal.sol` documents open access. [5](#0-4) 

**unsupported assumption checked:** The critical assumption — that the roller's own implementation also lacks a caller check — cannot be confirmed from source alone. This is precisely why local proof is required before promotion.

## Local Proof Required

**test type:** Foundry fork test against BSC mainnet (proxy `0xe2ce6ab80874fa9fa2aae65d277dd6b8e65c9de0`, implementation `0xeb6c62d1885c421797d0c494e693c90d4b54125a`)

**test file to add:** `test/fork/SetTokenBeneficiaryAccessControl.t.sol`

**test setup:**
1. Fork BSC at a recent block (≥ 108382650).
2. Identify a live token address from a recent `TokenCreated` event where `_tokenBeneficiaries[token]` is set (query via `getTokenV8Safe` or event logs).
3. Use a fresh EOA (`attacker`) that has no roles on the Portal.

**call sequence:**
```solidity
vm.prank(attacker);
portal.setTokenBeneficiary(token, attacker);
```

**expected assertion (if vulnerable):**
```solidity
assertEq(portal.getTokenBeneficiary(token), attacker);
// or assert BeneficiaryChanged event emitted with attacker address
```

**expected assertion (if protected):**
```solidity
vm.expectRevert(); // role check or caller check in roller
portal.setTokenBeneficiary(token, attacker);
```

**failure condition:** If the call succeeds and `_tokenBeneficiaries[token]` is updated to `attacker` without `DEFAULT_ADMIN_ROLE`, the finding is confirmed as a reward-extraction vulnerability. If it reverts, the roller has its own guard and the finding is rejected.
