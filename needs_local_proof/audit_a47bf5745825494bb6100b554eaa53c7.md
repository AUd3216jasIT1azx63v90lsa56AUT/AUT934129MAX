## Attacker Path
**preconditions:**
- A token exists on the bonding curve with an ERC20 quote token (e.g. USDT) and `nativeToQuoteSwapEnabled = true` for that quote token configuration.
- The BNB/USDT pool used by `MULTI_DEX_ROUTER` has sufficient depth to accept the swap but will return fewer USDT than a naive `msg.value × spot_price` estimate due to price impact or fee.

**attacker-controlled inputs:**
- `params.inputToken = address(0)` (native BNB), `params.outputToken = <bonding-curve token>`, `params.inputAmount = msg.value`, `params.minOutputAmount = 0` (or any value below actual output).

**call sequence:**
1. Attacker calls `swapExactInput{value: X BNB}(...)` on the Portal proxy.
2. Portal delegates to `PORTAL_TRADE_V2`.
3. `PORTAL_TRADE_V2` detects `inputToken == address(0)` and `quoteToken != address(0)` → triggers nativeToQuoteSwap path.
4. `MULTI_DEX_ROUTER` swaps X BNB → Y USDT where Y < X × spot_price (slippage/fee).
5. **Hypothesised bug:** reserve is updated by `X × spot_price` (or `msg.value` converted at some pre-swap rate) rather than the actual Y USDT received.
6. `state.reserve` is now overstated by `(X × spot_price) − Y`.
7. Attacker (or a colluding seller) calls `swapExactInput` to sell the bonding-curve token back, receiving USDT computed against the inflated reserve, extracting more USDT than was actually deposited.

## Why Existing Checks Fail
The `minOutputAmount` slippage guard on `ExactInputParams` protects the *caller's token output* (how many bonding-curve tokens they receive), not the reserve accounting integrity. [4](#0-3) 

If the `PORTAL_TRADE_V2` facet passes `msg.value` (or a pre-swap quote) rather than the router's actual return value to `_setTokenReserve`, no existing on-chain check reconciles `state.reserve` against `IERC20(quoteToken).balanceOf(address(this))` after the swap. The `_setTokenReserve` primitive writes whatever value it is given without any balance-check guard. [2](#0-1) 

**Critical limitation:** The `PORTAL_TRADE_V2` facet source is not present in the indexed `src/` directory — `swapExactInput` in `Portal.sol` is a pure dispatcher. The reserve-update logic that determines whether the actual router output or a pre-swap estimate is used **cannot be confirmed from the available source alone**. This is why local proof is required. [5](#0-4) 

## Rejection Checks
**expected behavior checked:** Normal slippage on the user's output tokens is expected; using `msg.value` instead of actual USDT received for reserve accounting is not expected behavior — it would be a concrete accounting error.

**prior report checked:** Not found in available source comments or NatSpec.

**README/NatSpec checked:** The `swapExactInput` NatSpec documents the nativeToQuoteSwap scenario but does not specify which value is used for the reserve update. [6](#0-5) 

**unsupported assumption checked:** The theory does not require oracle failure, malicious token owner, or admin compromise — it requires only that the reserve update uses a value other than the actual ERC20 amount received from the router, which is a verifiable implementation detail.

## Local Proof Required
**test type:** Foundry fork test on BSC mainnet (block ≥ 108382650).

**test file to add:** `test/fork/NativeToQuoteSwapReserveAccounting.t.sol`

**test setup:**
1. Fork BSC at latest block; use Portal proxy `0xe2ce6ab80874fa9fa2aae65d277dd6b8e65c9de0`.
2. Identify (or create) a bonding-curve token whose `quoteTokenAddress` is USDT and `nativeToQuoteSwapEnabled = true`.
3. Record `state.reserve` (via `getTokenV8`) and `IERC20(USDT).balanceOf(portal)` before the swap.
4. Call `swapExactInput{value: 1 ether}(inputToken=address(0), outputToken=<token>, inputAmount=1e18, minOutputAmount=0, permitData="")`.
5. Record `state.reserve` and `IERC20(USDT).balanceOf(portal)` after the swap.
6. Compute `deltaReserve = state.reserve_after − state.reserve_before` and `deltaBalance = USDT.balanceOf(portal)_after − USDT.balanceOf(portal)_before`.

**expected assertion:**
```
assertEq(deltaReserve, deltaBalance, "reserve must equal actual USDT received");
```

**failure condition:** If `deltaReserve > deltaBalance` (reserve credited more than actual USDT received), the invariant is broken and the finding is confirmed. A subsequent sell against the inflated reserve should then return more USDT than was deposited, proving fund extraction.

    function _setTokenReserve(address token, uint128 reserve) internal {
        if (token != address(uint160(uint256(uint160(token))))) revert DirtyBits();
        uint256 slot;
        assembly ("memory-safe") {
            mstore(0x0, token)
            mstore(0x20, _packedTokenStates.slot)
            slot := keccak256(0x0, 0x40)
        }
        uint256 packed;
        assembly ("memory-safe") {
            packed := sload(slot)
        }
        uint8 header = uint8(packed);
        if (header != PACKED_TOKEN_STATE_HEADER) {
            packed = (packed & ~(uint256(type(uint96).max) << 56)) | (uint256(uint96(reserve)) << 56);
            assembly ("memory-safe") {
                sstore(slot, packed)
            }
        } else {
            uint256 slot1 = slot + 1;
            uint256 slot1val;
            assembly ("memory-safe") {
                slot1val := sload(slot1)
            }
            slot1val = (slot1val & ~uint256(type(uint128).max)) | uint256(reserve);
            assembly ("memory-safe") {
                sstore(slot1, slot1val)
            }
        }
    }
