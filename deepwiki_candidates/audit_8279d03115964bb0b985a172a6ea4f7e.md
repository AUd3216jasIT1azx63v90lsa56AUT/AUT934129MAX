## Attacker Path

**preconditions:**
- A token is in `TokenStatus.Tradable` with `circulatingSupply` just below `dexSupplyThresh`.
- The migration path in `PORTAL_TRADE_V2` makes an external call (to a migrator/locker/token contract) **after** the threshold is crossed but **before** `_setTokenStatus(DEX)` is committed, OR the migrator itself makes an external call that the attacker controls (e.g., a token transfer callback to an attacker-controlled recipient contract).

**attacker-controlled inputs:**
- `ExactInputParams` with `outputToken` = target token, `inputToken` = native/quote, `recipient` = attacker contract that implements a receive/callback.

**call sequence:**
1. Attacker calls `swapExactInput` with exactly enough input to push `circulatingSupply >= dexSupplyThresh`.
2. Inside `PORTAL_TRADE_V2`, the migration is triggered. If `_setTokenStatus(DEX)` has not yet been written when an external call occurs (e.g., token transfer to recipient, migrator callback), the attacker's contract re-enters `swapExactInput`.
3. The re-entrant call hits `_getSwapImplementation`: status is still `Tradable` → routes back to `PORTAL_TRADE_V2`.
4. `PORTAL_TRADE_V2` processes the buy at bonding-curve price against the same (not-yet-migrated) reserve.
5. Outer call resumes, migrates liquidity calculated from the pre-reentrant reserve — but `circulatingSupply` is now higher than expected, leaving the DEX pool underfunded relative to outstanding tokens.

## Why Existing Checks Fail

The only guard on `swapExactInput` is the `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)` modifier, which checks a feature flag, not reentrancy. [1](#0-0) 

The `_getSwapImplementation` routing reads `state.status` from storage at the moment of the re-entrant call. If `_setTokenStatus(DEX)` has not yet been executed in the outer call frame, the re-entrant call sees `Tradable` and is routed to `PORTAL_TRADE_V2` — the bonding-curve handler — not to `PORTAL_DEX_ROUTER`. [4](#0-3) 

The `_setTokenStatus` write is a simple SSTORE with no mutex: [3](#0-2) 

## Rejection Checks

**expected behavior checked:** Buying at bonding-curve price after migration is triggered is not documented as expected behavior; the invariant is that no bonding-curve trades occur after the DEX threshold is crossed.

**prior report checked:** Not determinable from available source; no NatSpec or README in-scope files documents this as a known issue.

**README/NatSpec checked:** No NatSpec on `swapExactInput` or migration path addresses reentrancy.

**unsupported assumption checked:** The critical assumption — that `PORTAL_TRADE_V2` makes an external call before committing `_setTokenStatus(DEX)` — **cannot be verified** because `PORTAL_TRADE_V2` is an immutable facet address not present in this repository (`src/` contains only `Portal.sol`, `PortalBase.sol`, `PortalCommon.sol`, and interfaces). This is the primary reason the verdict is NEEDS_LOCAL_PROOF rather than HIGH_CONFIDENCE_CANDIDATE.

## Local Proof Required

**test type:** Foundry fork test or unit test with mock `PORTAL_TRADE_V2` stub.

**test file to add:** `test/ReentrancyMigration.t.sol`

**test setup:**
1. Deploy Portal with a mock `PORTAL_TRADE_V2` that: (a) checks `circulatingSupply >= dexSupplyThresh`, (b) transfers tokens to recipient (triggering attacker callback), (c) then calls `_setTokenStatus(DEX)`, (d) then calls migrator.
2. Deploy attacker contract whose `receive`/`onERC20Received` re-enters `swapExactInput` with a small buy.
3. Set token state to `Tradable` with `circulatingSupply` one unit below threshold.
4. Call `swapExactInput` from attacker contract with exactly the threshold-crossing amount.

**expected assertion:**
- If vulnerable: re-entrant `swapExactInput` succeeds, attacker receives tokens at bonding-curve price, final `circulatingSupply` exceeds the amount accounted for in the migrated liquidity.
- If protected: re-entrant call reverts with `TokenNotTradable` or equivalent.

**failure condition:** Re-entrant call succeeds AND attacker's token balance after migration exceeds what the DEX pool was funded to cover — confirming fund extraction via under-provisioned DEX liquidity.

```text
    function _getSwapImplementation(address inputToken, address outputToken)
        internal
        view
        returns (address implementation)
    {
        // Determine if this is a buy or sell
        bool isBuy = _quoteTokenConfigurations[inputToken].enabled == 1;
        address baseToken = isBuy ? outputToken : inputToken;

        // Get token state to check status
        PackedTokenStateV2 memory state = _getTokenState(baseToken);

        if (state.status == TokenStatus.DEX) {
            // Token is listed on DEX, use PortalDexRouter
            return PORTAL_DEX_ROUTER;
        } else if (state.status == TokenStatus.Tradable) {
            // Token is still in bonding curve, use PortalTradeV2
            return PORTAL_TRADE_V2;
        } else {
            // Invalid or other status, will fail in implementation
            return PORTAL_TRADE_V2; // Let PortalTradeV2 handle the error
        }
    }
```

```text
    function _setTokenStatus(address token, IPortalTypes.TokenStatus status) internal {
        if (token != address(uint160(uint256(uint160(token))))) revert DirtyBits();
        uint256 slot;
        assembly ("memory-safe") {
            mstore(0x0, token)
            mstore(0x20, _packedTokenStates.slot)
            slot := keccak256(0x0, 0x40)
        }
        uint256 packed;
        assembly ("memory-safe") {
            packed := sload(slot)
        }
        uint8 header = uint8(packed);
        if (header != PACKED_TOKEN_STATE_HEADER) {
            packed = (packed & ~uint256(0xff)) | uint8(status);
            assembly ("memory-safe") {
                sstore(slot, packed)
            }
        } else {
            packed = (packed & ~(uint256(0xff) << 8)) | (uint256(uint8(status)) << 8);
            assembly ("memory-safe") {
                sstore(slot, packed)
            }
        }
    }
```
