## Attacker Path

**preconditions:**
- A token is in `TokenStatus.Tradable` with `circulatingSupply` close to `dexSupplyThresh`.
- The attacker controls a contract that can receive a native-token callback (e.g., BNB refund or migrator callback) during the migration flow.

**attacker-controlled inputs:**
- `ExactInputParams.inputAmount` sized to push `circulatingSupply` exactly to `dexSupplyThresh`, triggering migration inside `PORTAL_TRADE_V2`.

**call sequence:**
1. Attacker calls `swapExactInput` (buy) with `inputAmount` that crosses `dexSupplyThresh`.
2. Inside the delegatecall to `PORTAL_TRADE_V2`, the migration path calls `luanchToDEX(token)` on the external migrator **before** calling `_setTokenStatus(token, TokenStatus.DEX)` (the hypothesised CEI violation).
3. The migrator (or any intermediate external call) triggers a callback to the attacker's contract.
4. Attacker re-enters `Portal.swapExactInput` (buy) for the same token.
5. `_getSwapImplementation` reads `state.status` — still `Tradable` because `_setTokenStatus(DEX)` has not yet executed — and returns `PORTAL_TRADE_V2`.
6. The bonding-curve buy succeeds: attacker receives tokens at the (now-stale) bonding-curve price before the DEX pool is funded.
7. Outer call resumes, sets status to DEX, and provides liquidity; attacker holds tokens bought at below-graduation price.

## Why Existing Checks Fail

The **only** guard against this scenario is the `state.status` read in `_getSwapImplementation`. [5](#0-4) 

This guard is effective **only if** `_setTokenStatus(DEX)` is called **before** any external call in the migration path (CEI). If the migrator or any intermediate contract is called first, the status is still `Tradable` at re-entry time, and `_getSwapImplementation` returns `PORTAL_TRADE_V2`, routing the re-entrant buy through the bonding curve.

There is no `nonReentrant` modifier, no lock variable, and no other mechanism in the available source that would block a re-entrant `swapExactInput` call while the migration external call is in flight. [6](#0-5) 

**Critical unknown:** The `PORTAL_TRADE_V2` implementation (the delegatecall target that contains the actual migration trigger, the `circulatingSupply >= dexSupplyThresh` check, and the call to `luanchToDEX`) is **not present** in this repository's source files. The `source_identity.source_files` list confirms only `src/Portal.sol`, `src/PortalBase.sol`, `src/PortalCommon.sol`, and interface/library files are available. Whether `PORTAL_TRADE_V2` follows CEI (sets status before external call) or violates it (calls migrator before setting status) cannot be determined from the available source alone.

## Rejection Checks

**expected behavior checked:** The `_getSwapImplementation` status dispatch is the intended guard; the question is whether it is set up in time. Not expected behavior to allow bonding-curve buys post-migration-trigger.

**prior report checked:** No prior report found in available source or NatSpec.

**README/NatSpec checked:** `IPortalMigrator.luanchToDEX` NatSpec states it "may be delegatecalled from a payable function" but gives no ordering guarantee relative to `_setTokenStatus`. [7](#0-6)

If CEI is respected (status set to DEX before any external call), the re-entrant `swapExactInput` is routed to `PORTAL_DEX_ROUTER` and the bonding-curve path is unreachable. This assumption must be verified by inspecting or decompiling the live `PORTAL_TRADE_V2` bytecode at the implementation address.

        internal
        view

    {
        // Determine if this is a buy or sell
        bool isBuy = _quoteTokenConfigurations[inputToken].enabled == 1;

        // Get token state to check status

        if (state.status == TokenStatus.DEX) {
            // Token is listed on DEX, use PortalDexRouter
            return PORTAL_DEX_ROUTER;

            // Token is still in bonding curve, use PortalTradeV2
            return PORTAL_TRADE_V2;
        } else {
            // Invalid or other status, will fail in implementation
            return PORTAL_TRADE_V2; // Let PortalTradeV2 handle the error
        }
    }
