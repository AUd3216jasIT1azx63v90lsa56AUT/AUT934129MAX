
The Portal-level entry point carries **only** the `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)` modifier. [1](#0-0)  Any role check must therefore live inside the `PORTAL_DEX_ROUTER` delegatecall target. The `PORTAL_DEX_ROUTER` implementation source is **not present** in this repository (the glob of `src/**/*.sol` returns no `PortalDexRouter.sol`). [2](#0-1)  The `_dexPools` mapping that would be written is declared in `PortalBase` storage and is directly accessible via delegatecall. [3](#0-2) 

The routing decision in `swapExactInput` delegates to `PORTAL_DEX_ROUTER` whenever `state.status == TokenStatus.DEX`, meaning any DEX-graduated token swap reads `_dexPools[token]` inside that same implementation. [4](#0-3) 

## Attacker Path
**preconditions:**
- Protocol is not halted (`bitFlags & 1 == 1`)
- Target token has `status == TokenStatus.DEX` (already graduated to DEX)
- `PORTAL_DEX_ROUTER` implementation has no role check on `updateTokenPoolInfo` (unverified — source not in repo)

**attacker-controlled inputs:**
- `token`: address of any DEX-graduated token
- `poolInfo.pool`: attacker-controlled contract that returns inflated `amountOut`

**call sequence:**
1. `Portal.updateTokenPoolInfo(token, {pool: attackerPool, fee: ..., poolType: ..., ...})`
   → delegatecall to `PORTAL_DEX_ROUTER` → writes `_dexPools[token].pool = attackerPool`
2. `Portal.swapExactInput({inputToken: quoteToken, outputToken: token, inputAmount: X, minOutputAmount: 0, ...})`
   → `_getSwapImplementation` returns `PORTAL_DEX_ROUTER` (status == DEX)
   → delegatecall to `PORTAL_DEX_ROUTER` → reads `_dexPools[token]` → routes swap through `attackerPool`
   → `attackerPool` returns inflated output; attacker receives excess tokens or BNB

## Why Existing Checks Fail
The only guard at the Portal entry point is `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)`, which is a liveness switch, not an access-control check. [5](#0-4)  The live context lists `updateTokenPoolInfo` under `role_or_config_entrypoints`, indicating the *intent* is privileged access, but that intent is not enforced at the Portal layer. [6](#0-5)  Whether the `PORTAL_DEX_ROUTER` implementation enforces a role check (e.g., `onlyRole(DEFAULT_ADMIN_ROLE)`) cannot be determined from the available source — this is the critical unknown.

```text
    function _getSwapImplementation(address inputToken, address outputToken)
        internal
        view
        returns (address implementation)
    {
        // Determine if this is a buy or sell
        bool isBuy = _quoteTokenConfigurations[inputToken].enabled == 1;
        address baseToken = isBuy ? outputToken : inputToken;

        // Get token state to check status
        PackedTokenStateV2 memory state = _getTokenState(baseToken);

        if (state.status == TokenStatus.DEX) {
            // Token is listed on DEX, use PortalDexRouter
            return PORTAL_DEX_ROUTER;
        } else if (state.status == TokenStatus.Tradable) {
            // Token is still in bonding curve, use PortalTradeV2
            return PORTAL_TRADE_V2;
        } else {
            // Invalid or other status, will fail in implementation
            return PORTAL_TRADE_V2; // Let PortalTradeV2 handle the error
        }
    }
```
