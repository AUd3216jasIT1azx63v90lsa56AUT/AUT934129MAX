## Attacker Path
**preconditions:**
- `bitFlags` has `CB_BIT_MASK_GLOBAL_SWITCH` set (portal not halted)
- A token `T` has `status == TokenStatus.DEX` (already migrated, so `_dexPools[T]` is set to the real pool)
- `PORTAL_DEX_ROUTER` implementation does **not** enforce `DEX_ROUTER_MANAGER_ROLE` on `updateTokenPoolInfo`

**attacker-controlled inputs:**
- `token = T` (any DEX-listed token address)
- `poolInfo.pool = attacker_fake_pool` (attacker-controlled contract)
- `poolInfo.poolType`, `poolInfo.fee`, etc. set to match expected V2/V3 type

**call sequence:**
1. Attacker calls `Portal.updateTokenPoolInfo(T, {pool: attacker_fake_pool, ...})`
2. `onlyIfBitFlagsSet` passes (portal is live)
3. `_delegateToImpl(PORTAL_DEX_ROUTER)` executes in Portal's storage context
4. If no role check in `PORTAL_DEX_ROUTER`: `_dexPools[T]` is overwritten with `attacker_fake_pool`
5. Subsequent `swapExactInput` calls for token `T` route through `attacker_fake_pool`, which can steal quote tokens from swappers

## Why Existing Checks Fail
`onlyIfBitFlagsSet` only checks that the global feature flag is non-zero â€” it is not an `AccessControl` role check and does not verify `msg.sender` has `DEX_ROUTER_MANAGER_ROLE`. [4](#0-3) 

The `DEX_ROUTER_MANAGER_ROLE` is declared in the interface but its enforcement location is the unverified `PORTAL_DEX_ROUTER` implementation. If that implementation omits the `hasRole` check (or uses a different guard that can be bypassed), the Portal.sol layer provides no backstop. [5](#0-4)
