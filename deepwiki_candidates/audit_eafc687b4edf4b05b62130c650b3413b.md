## Attacker Path
**preconditions:**
- Any DEX-status token exists (i.e., a token that has been launched to DEX and has a `_dexPools` entry)
- Global bit flag is set (portal not halted)
- Attacker deploys a malicious contract that mimics a V4/PCS_INFINITY_CL pool interface but returns manipulated swap amounts or steals input tokens

**attacker-controlled inputs:**
- `token`: address of a legitimate DEX-launched token
- `poolInfo.pool`: attacker's malicious pool address
- `poolInfo.poolType`: `V4` or `PCS_INFINITY_CL`

**call sequence:**
1. Attacker calls `Portal.updateTokenPoolInfo(victimToken, {pool: maliciousPool, poolType: V4, ...})` â€” succeeds because only `onlyIfBitFlagsSet` is checked
2. Victim calls `Portal.swapExactInput(...)` for `victimToken`
3. `PORTAL_DEX_ROUTER` reads `_dexPools[victimToken]`, finds `poolType == V4`, routes swap to `maliciousPool`
4. Malicious pool receives victim's input tokens and returns a manipulated/zero output amount
5. Victim loses input tokens; attacker captures them via the malicious pool

## Why Existing Checks Fail
The proxy-level guard `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)` only checks that the portal is not globally halted â€” it is not an access-control check. [3](#0-2) 

The `DEX_ROUTER_MANAGER_ROLE` restriction documented in the interface NatSpec (`// @dev can only be called by DEX_ROUTER_MANAGER_ROLE roles`) is never translated into an `onlyRole(DEX_ROUTER_MANAGER_ROLE)` modifier or `hasRole` check in any implementation file. A repo-wide grep for `DEX_ROUTER_MANAGER_ROLE` returns zero hits outside `src/interfaces/IPortal.sol`. [4](#0-3)
