## Attacker Path

**preconditions:**
- Portal is not halted (global bit flag is set, i.e. `bitFlags & 1 == 1`)
- A target token has already been launched to DEX (status = `TokenStatus.DEX`), so `_dexPools[token]` holds a legitimate pool address
- The `PORTAL_DEX_ROUTER` delegatecall target does NOT enforce `DEX_ROUTER_MANAGER_ROLE` (unconfirmed — see below)

**attacker-controlled inputs:**
- `token`: address of any DEX-listed Portal token
- `poolInfo.pool`: attacker-controlled fake pool contract

**call sequence:**
1. Attacker deploys a fake pool contract that accepts token transfers and returns attacker-controlled amounts
2. Attacker calls `Portal.updateTokenPoolInfo(token, PackedDexPool{pool: fakePool, ...})`
3. If no role check exists in `PORTAL_DEX_ROUTER`, `_dexPools[token]` is overwritten with `fakePool`
4. Any subsequent `swapExactInput` / `swapExactInputV3` call for that DEX-listed token routes through `PORTAL_DEX_ROUTER`, which reads `_dexPools[token]` and interacts with `fakePool`
5. Attacker's fake pool drains quote tokens or launched tokens from the swap [4](#0-3) [5](#0-4) 

## Why Existing Checks Fail

The `onlyIfBitFlagsSet(CB_BIT_MASK_GLOBAL_SWITCH)` modifier only checks whether the global pause bit is set. [6](#0-5) 

It does not check any role. The role check is supposed to live inside the `PORTAL_DEX_ROUTER` delegatecall target. However:

1. A `grep` for `DEX_ROUTER_MANAGER_ROLE` across the entire repository returns **only two matches, both in `src/interfaces/IPortal.sol`** — one in the NatSpec comment and one as a constant declaration. No match appears in any implementation file.
2. A glob search for `src/**/*PortalDex*` returns **no results** — the `PORTAL_DEX_ROUTER` implementation source is not present in the indexed repository files.
3. The `source_identity.source_files` for the live implementation (`0xeb6c62d1885c421797d0c494e693c90d4b54125a`) lists only `src/Portal.sol`, `src/PortalBase.sol`, `src/PortalCommon.sol`, and interface/library files — no `PortalDexRouter.sol`. [3](#0-2) 

Because the `PORTAL_DEX_ROUTER` implementation is a separate deployed contract whose source is not indexed here, it is **not possible to confirm from source alone** whether the role check is present or absent in the delegatecall target. This is the blocking uncertainty.
